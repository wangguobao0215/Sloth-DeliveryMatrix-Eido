# Progress Tracker - 进度追踪能力模块

> 模块类型: 能力模块 (Capability Module)
> 调用方式: 被角色模块路由调用，或用户直接请求"看进度""项目进展""里程碑情况"

## 1. 模块职责

本模块负责对项目进度进行量化分析、可视化呈现和趋势预测。支持单项目和跨项目两种模式。

## 2. 里程碑完成度分析

### 2.1 里程碑状态判定

对每个里程碑按以下规则判定状态：

| 状态 | 判定规则 | 标记 |
|------|---------|------|
| 已完成 | 实际完成日 ≤ 计划完成日 | ✅ 按时 |
| 延迟完成 | 实际完成日 > 计划完成日 | ⚠️ 延迟 N 天 |
| 进行中-正常 | 今日 ≤ 计划完成日，且进展符合预期 | 🔵 正常 |
| 进行中-预警 | 今日 ≤ 计划完成日，但进展落后于预期 | 🟡 预警 |
| 进行中-告警 | 今日 > 计划完成日，且尚未完成 | 🔴 逾期 |
| 未开始 | 计划开始日 > 今日 | ⏳ 待开始 |

### 2.2 里程碑完成度计算

```
里程碑完成度 = 已完成里程碑数 / 总里程碑数 × 100%

阶段完成度 = 当前阶段内已完成任务数 / 当前阶段总任务数 × 100%

整体进度估算:
  - 如果有任务级数据: 按任务完成率加权计算
  - 如果只有里程碑数据: 按里程碑完成率近似
  - 如果数据不足: 标注"估算"并说明依据
```

## 3. SPI（进度绩效指数）计算

### 3.1 SPI 计算公式

```
SPI = EV / PV

其中:
  EV (Earned Value, 挣值) = 实际完成工作量的计划价值
  PV (Planned Value, 计划值) = 截至当前时间应完成工作量的计划价值

简化计算（当缺乏详细工作量数据时）:
  SPI ≈ 实际完成的里程碑进度百分比 / 计划应完成的里程碑进度百分比

示例:
  - 项目计划90天，已过60天（计划应完成66.7%）
  - 实际完成进度 55%
  - SPI = 55% / 66.7% = 0.82
```

### 3.2 SPI 解读

| SPI 范围 | 含义 | 建议行动 |
|---------|------|---------|
| > 1.1 | 进度超前 | 检查是否有范围遗漏或质量隐患 |
| 1.0 - 1.1 | 进度正常 | 维持当前节奏 |
| 0.9 - 1.0 | 轻微落后 | 关注并微调 |
| 0.8 - 0.9 | 明显落后 | 启动赶工或调整计划 |
| < 0.8 | 严重落后 | 需升级处理，重新评估项目计划 |

### 3.3 SPI 趋势分析

连续追踪 SPI 变化，输出趋势判断：

```
SPI 趋势判断:
  - 连续3周 SPI > 1.0 且稳定: 项目健康，保持节奏
  - SPI 从 >1.0 逐渐下降至 <1.0: 进度在恶化，需关注
  - SPI 持续 <0.9: 项目可能需要重新规划
  - SPI 从 <0.9 逐渐回升至 >0.95: 纠偏措施生效
```

## 5. 两种运行模式

### 5.1 单项目模式 (single_project)

聚焦单个项目的深度进度分析。输出包含：
1. 进度摘要（文本）
2. 里程碑详情表
3. SPI 分析与趋势
4. 进度预测（按当前速率，预计完成日期）
5. 风险提示（进度相关）

**进度预测公式**:
```
预计完成日 = 今日 + 剩余工作量 / 当前速率
当前速率 = 最近2周平均每日完成工作量
如果预计完成日 > 计划完成日: 标注延期风险
```

### 5.2 跨项目模式 (cross_project)

面向 Director 的全景视图。输出包含：

```
## 项目健康度排名

| 排名 | 项目 | 阶段 | 总进度 | SPI | 健康度 | 趋势 | 关注点 |
|------|------|------|--------|-----|--------|------|--------|
| 1 | CRM二期 | 测试 | 85% | 1.02 | 🟢 健康 | → | 按计划推进 |
| 2 | 数据中台 | 开发 | 45% | 0.95 | 🟡 关注 | ↓ | SPI连降2周 |
| 3 | 智慧园区 | 开发 | 35% | 0.78 | 🔴 告警 | ↓ | 严重落后 |

### 整体交付态势
- 3个在管项目，1个健康，1个需关注，1个告警
- 整体交付健康度: 中等偏下
- 本周关键事件: CRM二期进入UAT，智慧园区需决策是否调整计划
```

## 6. 数据不足时的处理

当项目数据不完整时，按以下策略降级处理：

| 数据级别 | 可用数据 | 分析能力 |
|---------|---------|---------|
| 完整 | 任务级数据 + 工时 | 完整 SPI + 精确预测 |
| 中等 | 里程碑 + 阶段进度 | 简化 SPI + 粗略预测 |
| 基础 | 仅里程碑 | 里程碑完成度 + 定性判断 |
| 极简 | 用户口述 | 定性状态标注，建议补充数据 |

无论数据级别如何，都必须明确标注数据来源和分析的置信度。

## 7. 成本追踪（可选）

当项目状态中包含预算信息时，自动计算以下成本指标：

| 指标 | 公式 | 说明 |
|------|------|------|
| **CPI** (成本绩效指数) | EV / AC | >1 表示成本节约，<1 表示超支 |
| **EAC** (完工估算) | BAC / CPI | 按当前趋势预测最终成本 |
| **VAC** (完工偏差) | BAC - EAC | 正值=节约，负值=超支 |
| **消耗率** | AC / BAC × 100% | 预算消耗百分比 |

- **EV** (Earned Value): 已完成工作的预算值 = 完成率 × BAC
- **AC** (Actual Cost): 实际花费（从工时 × 人天单价计算）
- **BAC** (Budget at Completion): 项目总预算

### 成本预警阈值
- CPI < 0.9 → 红色警告："成本超支超过 10%，建议立即审查"
- CPI 0.9-1.0 → 黄色提醒
- CPI > 1.0 → 绿色正常

若项目未设置预算数据，跳过成本章节，标注 `[未设置预算，如需成本追踪请先录入预算信息]`。
