# Client Communicator - 客户沟通能力模块

> 模块类型: 能力模块 (Capability Module)
> 调用方式: 用户请求"写邮件给客户""通知客户""客户沟通"或需要对外沟通时触发

## 1. 模块职责

本模块负责生成面向客户的各类沟通内容，确保专业性、准确性和恰当的语气。支持中英文双语输出。

## 2. 沟通类型

| 沟通类型 ID | 名称 | 使用场景 | 语气基调 | 典型长度 |
|------------|------|---------|---------|---------|
| `status_update_email` | 项目状态更新邮件 | 定期向客户汇报进展 | 专业、积极 | 300-500字 |
| `escalation_email` | 问题升级邮件 | 向客户通报问题/风险 | 客观、解决方案导向 | 400-600字 |
| `change_request` | 变更请求沟通 | 通知客户需求变更的影响 | 严谨、数据支撑 | 500-800字 |
| `milestone_notification` | 里程碑完成通知 | 阶段交付物完成通知 | 积极、成果导向 | 200-400字 |
| `meeting_invitation` | 会议邀请 | 发起与客户的会议 | 简洁、明确 | 100-200字 |
| `acceptance_invitation` | 验收邀请 | 邀请客户进行验收 | 正式、详细 | 400-600字 |
| `delay_notification` | 延期通知 | 通知客户进度延迟 | 诚恳、有预案 | 400-600字 |

## 3. 语气调整规则

### 3.1 语气选择矩阵

根据沟通内容的性质自动调整语气：

| 内容性质 | 语气 | 措辞特征 | 示例 |
|---------|------|---------|------|
| 正面进展 | 积极、自信 | "顺利完成""达成了""取得了阶段性成果" | "本周我们已顺利完成接口联调里程碑..." |
| 正常推进 | 专业、平实 | "按计划推进""正在进行中""预计按期" | "项目整体按计划推进，当前处于开发阶段..." |
| 问题通报 | 客观、解决方案导向 | "发现了""已采取措施""建议" | "在测试过程中发现了以下问题，我们已采取..." |
| 延期通知 | 诚恳、透明、有预案 | "对此我们深表歉意""调整后的计划""确保" | "由于XX原因，原计划的交付日期需要调整..." |
| 变更影响 | 严谨、数据化 | "经评估""影响范围""建议方案" | "经过详细评估，该变更将影响以下方面..." |

### 3.2 禁止用语

在客户沟通中，严禁使用以下表述：

| 禁止用语 | 原因 | 替代表述 |
|---------|------|---------|
| "我们加个班就行了" | 暴露内部管理问题 | "我们将调配额外资源确保进度" |
| "这不是我们的问题" | 推卸责任 | "经分析，该问题涉及XX环节，建议共同排查" |
| "应该没问题" | 不确定性 | "根据当前评估，预计可以按期完成" |
| "不可能实现" | 过于绝对 | "实现该需求需要额外XX人天，建议讨论优先级" |
| "我们人手不够" | 暴露内部短板 | "为确保交付质量，建议分阶段实施" |

### 3.3 坏消息传递原则

当需要向客户传递不好的消息（延期、问题、变更影响）时，遵循 STAR 原则：

```
S - Situation (情况): 客观描述当前状况，不回避
T - Timeline (时间线): 说明调整后的时间计划
A - Action (行动): 已采取和将要采取的补救措施
R - Request (请求): 需要客户配合的事项

绝不只通报问题而不给方案。
```

## 4. 沟通模板

### 4.1 状态更新邮件模板

```
主题: [项目名称] 第N周进展报告

尊敬的 [客户称呼]:

您好！以下是[项目名称]本周进展情况汇报。

一、本周进展
[用2-3个要点概述本周关键成果]

二、下周计划
[用2-3个要点概述下周重点工作]

三、需要协调的事项
[如有需要客户配合的事项，在此列出]

如有任何疑问，请随时联系我们。

此致
敬礼
[PM姓名] / [公司名称]项目组
```

### 4.2 问题升级邮件模板

```
主题: [项目名称] 关于[问题简述]的情况说明

尊敬的 [客户称呼]:

您好！关于[项目名称]中[问题简述]，现向您说明如下：

一、问题描述
[客观描述问题现象和发现时间]

二、影响分析
[说明对项目进度/范围/质量的影响]

三、已采取措施
[列出已执行的补救行动]

四、后续计划
[说明接下来的行动计划和预期恢复时间]

五、需要贵方支持
[如需客户配合，明确列出]

我们将持续跟进并及时向您汇报进展。对由此带来的不便，深表歉意。

此致
敬礼
[PM姓名] / [公司名称]项目组
```

### 4.3 延期通知邮件模板

```
主题: [项目名称] 关于[里程碑名称]交付时间调整的说明

尊敬的 [客户称呼]:

您好！经过内部评估，我们需要就[项目名称]的[里程碑名称]交付时间调整向您说明。

一、调整原因
[简要说明原因，客观不推诿]

二、调整方案
- 原计划完成日期: [日期]
- 调整后完成日期: [日期]
- 调整幅度: [N]个工作日

三、保障措施
[说明为确保新计划按时完成所采取的措施]

四、对后续里程碑的影响
[说明是否影响最终交付日期]

对此给贵方带来的不便，我们深表歉意。项目组将全力以赴确保调整后的计划按时达成。

如需进一步沟通，欢迎随时联系。

此致
敬礼
[PM姓名] / [公司名称]项目组
```

## 5. 双语支持

### 5.1 语言选择规则

```
语言选择:
1. 默认使用中文
2. 当 project.language == "en" 或 project.client_language == "en" 时使用英文
3. 当用户明确要求时切换语言
4. 某些场景需要中英双语（如外资客户的中国项目）
```

### 5.2 英文邮件注意事项

- 使用正式商务英语，避免口语化
- 主语使用 "we"（代表团队），避免 "I"
- 时态：已完成用过去式，计划用将来式，当前状态用现在进行时
- 缩写规则：首次出现使用全称 + 括号缩写，之后可用缩写

### 5.3 英文模板示例

```
Subject: [Project Name] Weekly Status Update - Week N

Dear [Client Name],

Please find below the status update for [Project Name] for the week ending [Date].

**Key Highlights**
- [Achievement 1]
- [Achievement 2]

**Upcoming Activities**
- [Plan 1]
- [Plan 2]

**Items Requiring Your Attention**
- [Item 1, if any]

Please feel free to reach out if you have any questions or concerns.

Best regards,
[PM Name]
[Company] Project Team
```

## 6. 沟通前自检

发送任何客户沟通前，执行以下检查：

```
## 客户沟通自检清单

### 内容准确性
- [ ] 项目名称、客户名称正确
- [ ] 日期、数据已核实
- [ ] 没有暴露内部敏感信息（如人员冲突、加班情况、成本信息）
- [ ] 技术术语已转换为客户能理解的语言

### 语气恰当性
- [ ] 语气与内容性质匹配（好消息积极、坏消息诚恳）
- [ ] 无禁止用语
- [ ] 坏消息已附带解决方案

### 行动明确性
- [ ] 需要客户配合的事项已明确列出
- [ ] 下一步行动和时间节点清晰
- [ ] 联系方式已提供

### 格式规范
- [ ] 邮件主题清晰且包含项目名称
- [ ] 段落简洁，重点突出
- [ ] 附件（如有）已标注
```
