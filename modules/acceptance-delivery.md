# Acceptance & Delivery - 验收交付能力模块

> 模块类型: 能力模块 (Capability Module)
> 调用方式: 用户请求"准备验收""验收材料""交付文档""结项"时触发

## 1. 模块职责

本模块负责项目验收和交付阶段的全流程支持，包括验收材料生成、交付物完整性检查、验收清单自动生成以及政府项目档案合规性检查。

## 3. 验收清单自动生成

### 3.1 生成逻辑

根据项目类型和需求文档自动生成验收清单：

```
生成流程:
1. 读取项目需求文档（BRD/SRS）中的功能列表
2. 为每个功能点生成至少1条验收标准
3. 根据 project.industry 添加行业特定验收项
4. 根据合同约定添加非功能性验收项（性能、安全等）
5. 输出完整验收清单
```

### 3.2 行业特定验收项

**政府项目额外验收项**：
- 等保合规性验证（如适用）
- 信创环境兼容性验证（国产化软硬件）
- 无障碍访问支持（如政府门户网站）
- 数据安全与权限管控
- 日志审计功能
- 档案归档完整性

**金融项目额外验收项**：
- 交易数据一致性验证
- 灾备切换验证
- 数据脱敏效果验证
- 操作审计日志完整性
- 报监管接口正确性

**SaaS 项目额外验收项**：
- 多租户隔离验证
- SLA 达标验证（可用性、响应时间）
- 数据导入/导出功能
- API 接口文档与实际一致性
- 升级回滚机制验证

## 5. 政府项目档案合规检查

### 5.1 政府信息化项目档案要求

政府项目结项时，需满足档案管理规范要求。本模块自动检查以下合规项：

```
## 档案合规检查: [项目名称]

### 立项阶段档案
- [ ] 项目建议书/可研报告
- [ ] 立项批复文件
- [ ] 招标/采购文件
- [ ] 中标通知书
- [ ] 合同签署文件

### 实施阶段档案
- [ ] 项目实施方案
- [ ] 需求确认文件（双方签字）
- [ ] 设计评审记录
- [ ] 变更申请及审批记录
- [ ] 会议纪要（关键会议）
- [ ] 阶段验收记录

### 验收阶段档案
- [ ] 验收测试方案
- [ ] 验收测试报告
- [ ] 第三方测评报告（如需）
- [ ] 验收报告（双方签字盖章）
- [ ] 培训记录

### 结项阶段档案
- [ ] 项目总结报告
- [ ] 资产移交清单
- [ ] 运维保障方案
- [ ] 保密协议（如涉密）

### 合规评估
- 必须项合规率: [X]%
- 缺失档案: [列表]
- 风险等级: [高/中/低]
- 建议: [补充措施及时间计划]
```

### 5.2 档案补充建议

当发现档案缺失时，按以下优先级建议补充：

| 优先级 | 档案类型 | 补充难度 | 建议 |
|--------|---------|---------|------|
| 紧急 | 合同/批复文件 | 高 | 与客户协调补签 |
| 紧急 | 验收报告（签字版）| 中 | 安排打印签字 |
| 高 | 需求确认文件 | 中 | 追溯邮件/会议记录整理 |
| 高 | 变更审批记录 | 中 | 从项目记录中整理 |
| 中 | 培训记录 | 低 | 补制签到表和培训材料 |
| 中 | 会议纪要 | 中 | 从项目记录中提取 |

## 7. 交接检查最终确认

在项目正式交付前，执行最终确认：

```
## 最终交付确认检查

### 系统层面
- [ ] 生产环境稳定运行 ≥ [N] 天
- [ ] 无阻塞性/严重缺陷
- [ ] 性能指标达标
- [ ] 数据备份机制已验证

### 文档层面
- [ ] 全部必须文档已提交
- [ ] 文档版本为最终版（与实际系统一致）
- [ ] 客户已确认接收

### 人员层面
- [ ] 培训已完成
- [ ] 运维交接已完成
- [ ] 保修/维护条款已确认

### 商务层面
- [ ] 验收报告已签字
- [ ] 发票/付款流程已启动
- [ ] 质保期条款已明确
```
