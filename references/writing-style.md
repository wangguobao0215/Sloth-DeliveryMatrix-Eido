# 写作风格指南 (Writing Style Guide)

> 规范 DeliveryMatrix 所有文档输出的语言风格、格式要求和表达标准，确保专业、清晰、可信。

---

## 一、总体原则

1. **专业但不官僚**：用简洁明确的语言传递信息，避免冗长的套话和空洞的表述。
2. **数据驱动**：每一个结论必须有数据支撑，不做无依据的判断。
3. **结论先行**：先说结论/建议，再给出依据和分析过程。
4. **面向行动**：每段输出最终指向具体的下一步行动，而非停留在描述层面。

---

## 二、结论先行写作模式

所有报告类输出遵循以下结构：

```
【结论/判断】一句话核心结论
【关键数据】支撑结论的 2-3 个关键指标
【分析说明】简要的推导过程
【建议行动】具体的下一步操作
```

**正确示例**：
> 本迭代进度存在延期风险。当前完成率 58%（目标 75%），主要瓶颈在支付模块联调，建议本周集中 2 名后端人力攻关。

**错误示例**：
> 经过对本迭代各项工作的全面梳理和深入分析，综合考虑各方面因素，我们认为当前进度情况需要引起一定程度的关注...

## 三、反 AI 味规则

### 禁用词汇及替代表达

以下词汇在正式输出中 **禁止使用**，必须替换为具体描述：

| 禁用词 | 问题 | 替代方式 |
|-------|------|---------|
| 赋能 | 空泛无实义 | 说清楚具体提供了什么能力或资源 |
| 闭环 | 滥用 | 用「完成」「跟进到位」「确认关闭」 |
| 抓手 | 官僚用语 | 用「关键措施」「核心手段」「具体做法」 |
| 打通 | 模糊 | 说清楚具体连通了什么系统/流程 |
| 沉淀 | 空泛 | 用「记录」「归档」「形成文档」 |
| 拉齐 | 流行但含混 | 用「对齐」「统一标准」「达成共识」 |
| 颗粒度 | 生造词感 | 用「细致程度」「拆分层级」「详细程度」 |
| 底层逻辑 | 滥用 | 用「核心原因」「基本原理」「根本逻辑」 |
| 顶层设计 | 官僚用语 | 用「整体规划」「总体方案」 |
| 链路 | 技术滥用 | 用「流程」「路径」「环节」 |
| 矩阵 | 非必要使用 | 用「表格」「分类」「对照」 |
| 生态 | 空泛 | 说清楚具体的系统、合作方或技术栈 |

### 补充禁用表达
- 「一定程度上」→ 给出具体程度
- 「相关工作」→ 说明具体是什么工作
- 「进一步优化」→ 说明优化什么、怎么优化、优化到什么目标
- 「持续推进」→ 说明推进什么、截止时间、负责人

## 四、数据表达规范

### 数字格式
- 百分比：保留 1 位小数，如 `85.3%`，不写 `85.33333%`。
- 金额：万元以下精确到元，万元以上用「万」为单位（如 `23.5 万元`），亿元以上用「亿」（如 `1.2 亿元`）。
- 人天：精确到 0.5 人天，如 `3.5 人天`。
- 时间：按场景选择精度——天级用日期，小时级用时刻，秒级罕见但需标注。

### 日期格式
- 正式文档：`2026年04月23日`
- 报表/表格：`2026-04-23`
- 口语化输出：`4月23日`（当年可省略年份）

### 趋势表达
- 上升用具体数值：「从 72% 提升至 85%」，不用「显著提升」。
- 下降同理：「从 15 个降至 8 个」，不用「大幅减少」。
- 对比必须给出两个数值和变化量。

---

## 五、中文排版规范

- 中文与英文/数字之间加半角空格：`本迭代完成 12 个用户故事`。
- 中文标点使用全角：逗号「，」、句号「。」、顿号「、」。
- 英文缩写首次出现时给出全称：`SPI（Schedule Performance Index，进度绩效指数）`。
- 列表项使用统一的编号风格：一级用 `1. 2. 3.`，二级用 `- `。
- 表格内容左对齐，数字右对齐。

---

## 六、政务/国企项目文档特殊规范

面向政务和国企客户的文档需额外满足：

### 文档编号
- 格式：`{公司缩写}-{项目编号}-{文档类型}-{版本号}`
- 示例：`SL-GY2026-SRS-V1.2`

### 密级标识
- 文档首页右上角标注密级：公开 / 内部 / 秘密 / 机密
- 默认密级为「内部」，由 PM 根据合同约定调整。

### 签字页
- 正式交付文档必须包含签字页，格式：

```
编制人：____________    日期：____________
审核人：____________    日期：____________
批准人：____________    日期：____________

甲方确认：____________  日期：____________
```

### 版本历史
- 所有正式文档首页必须包含版本历史表：

| 版本号 | 修改日期 | 修改人 | 修改内容摘要 |
|-------|---------|-------|------------|
| V1.0 | 2026年03月15日 | 张三 | 初始版本 |
| V1.1 | 2026年04月02日 | 李四 | 根据评审意见修改第 3 章 |
