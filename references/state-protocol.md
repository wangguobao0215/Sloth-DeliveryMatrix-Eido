# 状态管理协议 (State Protocol)

> 定义 DeliveryMatrix 全局与局部状态的存储、读写、版本控制和生命周期管理规则。

---

## 一、状态层级

状态按作用范围分为三个持久化层级，高层级状态对低层级可见：

| 层级 | 代号 | 作用域 | 典型内容 | 持久化 |
|------|------|--------|---------|--------|
| L0 | global | 全局 | 用户偏好、默认参数、技能配置 | 永久 |
| L1 | role | 角色级 | 当前角色工作上下文：最近路由、待办队列、输出摘要 | 永久 |
| L2 | project | 项目级 | 项目元数据、团队成员、里程碑、风险登记册（单文件存储） | 项目存活期 |

查询时按 L2 → L1 → L0 逐级回溯，低层级值覆盖高层级同名字段。会话级状态（L3）为 LLM 上下文中的临时数据，不持久化到文件。

## 二、文件路径规范

```
~/.qoderwork/data/Sloth-DeliveryMatrix-Eido/state/
  ├── global.yaml                        # L0 全局状态
  ├── role-state/
  │   └── active-context.yaml            # L1 当前角色工作上下文
  ├── projects/
  │   ├── {project_id}.yaml              # L2 单个项目完整状态（符合 project-state.schema.yaml）
  │   └── ...
  └── _backups/
      └── {project_id}.v{N}.yaml         # 历史版本备份
```

## 三、乐观锁与冲突检测

所有可写状态文件必须包含 `_version` 字段：

```yaml
_version: 17
_last_modified: "2026-04-20T14:30:00+08:00"
_modified_by: "session_abc123"
```

### 写入流程
1. 读取当前文件，获取 `_version = N`。
2. 执行修改逻辑，生成新内容。
3. 写入时校验文件中 `_version` 仍为 `N`，写入后设为 `N+1`。
4. 若 `_version` 已变（被其他会话修改），触发冲突处理。

### 冲突处理策略
- **字段级合并**：不同字段的修改自动合并。
- **同字段冲突**：保留两个版本，生成冲突报告交由用户决策。
- **changelog 追加**：每次写入在 `changelog.yaml` 追加变更记录。

---

## 四、备份策略

- 每次成功写入后，自动将旧版本存入 `state/_backups/` 目录。
- 保留最近 **20 个版本**，超出后按 FIFO 清理。
- 备份文件命名：`{project_id}.v{version}.yaml`
- 关键操作（删除成员、关闭里程碑）触发即时备份 + 操作日志。

```
state/_backups/
  ├── proj_001.v1.yaml
  ├── proj_001.v2.yaml
  ├── proj_002.v1.yaml
  └── ...
```

## 五、状态生命周期

```
create ──→ active ──→ archive
              ↑          │
              └── restore ┘
```

- **create**：首次创建状态文件，`_version = 1`。
- **active**：正常读写阶段，每次写入 `_version` 递增。
- **archive**：项目完结或长期不活跃（>90天无写入），状态文件移至 `_archived/`。
- **restore**：从归档恢复到活跃状态。

## 六、上下文窗口预算与状态裁剪

为控制注入 LLM 的上下文体积，遵循以下裁剪规则：

| 状态类型 | 最大 token 预算 | 裁剪策略 |
|---------|---------------|---------|
| L0 全局 | 200 tokens | 仅注入与当前意图相关的偏好字段 |
| L1 角色 | 300 tokens | 注入当前角色配置摘要 |
| L2 项目 | 1500 tokens | 注入项目摘要 + 当前迭代关键指标 |

裁剪优先级：先丢弃历史数据 → 再压缩描述字段 → 最后截断列表（保留最新 N 条）。会话级上下文由 LLM 对话历史管理，不计入文件裁剪预算。
