## 技能定位

Sloth-DeliveryMatrix-Eido 是 Sloth-Role-Eido 技能家族中的**软件交付智能体**，为项目经理、交付总监和技术顾问打造的数字副脑。

**核心价值**：
- **消灭重复劳动**——周报、会议纪要、风险登记表、进度快照等文档自动生成，PM 只需审阅和微调
- **放大感知半径**——从碎片信息中识别风险信号、资源瓶颈、需求偏移，比人早一步发现问题
- **保留决策主权**——所有输出是"决策参考"而非"替你做主"，最终判断永远属于你

**目标用户**：

| 角色 | 典型场景 |
|------|---------|
| 项目经理 / Scrum Master | 周报生成、会议纪要、风险评估、进度追踪、资源协调 |
| 交付总监 / PMO 负责人 | 跨项目全景仪表盘、资源池调度、成本控制、战略对齐 |
| 交付顾问 / QA 负责人 | 方法论指导、过程改进、质量审计、客户关系管理 |

## 安装与初始化

### 安装步骤

1. 在 QoderWork 中搜索并安装 `Sloth-DeliveryMatrix-Eido` 技能包
2. 首次使用时，系统自动进入引导流程

### 首次引导流程

当检测到没有已创建的项目时，系统会自动引导你完成项目初始化：

**第一步**：系统输出欢迎语，提示你创建第一个项目

**第二步**：回答 4 个简短问题（每次只问 1-2 个）：
1. 项目名称
2. 当前阶段——需求分析 / 设计开发 / 测试验收 / 上线运维
3. 项目类型——定制开发 / 产品实施 / 运维保障 / 咨询服务
4. 所属行业——政府/国企 / 金融 / 制造业 / 互联网/SaaS / 其他

**第三步**：系统自动创建项目状态文件，初始化完成

**第四步**：确认创建成功，展示可用功能

### 数据存储

所有项目数据存储在本地，路径为：

```
~/.qoderwork/data/Sloth-DeliveryMatrix-Eido/state/
├── global.yaml                    # 全局状态（活跃项目、角色）
├── role-state/
│   └── active-context.yaml        # 当前角色工作上下文
├── projects/
│   └── {project_id}.yaml          # 单个项目完整状态
└── _backups/                      # 自动备份
```

数据完全本地化，不依赖云端存储，跨会话可恢复。

## 核心功能详解

### 三种角色视角

| 角色 | 关注维度 | 切换触发词 |
|------|---------|-----------|
| **PM Commander**（默认） | 任务执行、进度追踪、风险管理、资源协调 | "切换到PM视角"、"回到项目经理"、"PM模式" |
| **Director Commander** | 项目组合管理、资源池调度、成本控制、战略对齐 | "切换到总监视角"、"总监模式"、"看全局" |
| **Consultant Commander** | 方法论指导、过程改进、质量审计、客户关系 | "切换到顾问视角"、"顾问模式"、"过程改进" |

三种角色共享同一套项目数据，切换角色时数据无缝继承，只有工作上下文和关注维度发生变化。

### 九大原子能力

#### R01 — 文档生成器

| 项目 | 说明 |
|------|------|
| **触发关键词** | 周报、写周报、本周总结、工作汇报、项目周报 |
| **触发模式** | `(本周\|这周\|上周).*(总结\|汇报\|周报)` |
| **输出格式** | 结构化标题 + 自然段落 + 表格 |
| **支持文档类型** | 周报、月报、BRD、SRS、测试报告、验收报告、评审材料、变更请求 |
| **模板系统** | 通用模板 (`templates/_base/`) / 政府模板 (`templates/gov/`) / SaaS 模板 (`templates/saas/`) / 金融模板 (`templates/finance/`) |

**使用示例**：
- "帮我写本周周报"
- "生成一份需求规格说明书"
- "写一份项目验收报告"

**使用示例**：
- "整理一下今天的会议纪要"（然后粘贴会议内容）

### 复合路由

除了九大单一能力外，系统还预定义了两条复合路由，自动编排多个能力：

| 复合路由 | 触发关键词 | 编排方式 | 子路由序列 |
|---------|-----------|---------|-----------|
| **项目复盘准备 (C01)** | 项目复盘、复盘准备、阶段总结 | 并行执行后合并 | R03 (风险) + R04 (进度) + R09 (数据) -> 合并为复盘报告 |
| **一键周报 (C02)** | 一键周报、完整周报、全量周报 | 顺序执行 | R09 (数据) -> R04 (进度) -> R03 (风险) -> R01 (周报) |

**使用示例**：
- "帮我准备项目复盘材料"
- "一键生成完整周报"

### 主动洞察

每次执行完主路由后，系统会自动扫描项目状态，附加最多 2 条主动提醒：

- **风险信号**——超期未处理的风险条目
- **里程碑预警**——7 天内到期且完成率不足 80% 的里程碑
- **资源冲突**——同一人在重叠时段被分配到多个任务

提醒以独立段落附在输出末尾，不打断主内容。可以说"别提醒了"关闭此功能。

## 工作流程指南

### 日常使用流程

```
1. 打开 QoderWork，系统自动加载上次的项目上下文
   ↓
2. 用自然语言描述你要做的事（写周报 / 整理纪要 / 评估风险等）
   ↓
3. 系统自动路由到对应模块，执行并输出结果
   ↓
4. 审阅输出，按需修改（"风险那部分改一下""再加一段关于测试的"）
   ↓
5. 项目状态自动更新，下次会话可恢复
```

### 多项目管理

支持同时管理多个项目：
- **新建项目**："新建一个项目"——进入引导流程创建新项目
- **切换项目**："切换到XX项目"——系统加载目标项目的状态
- **查看项目列表**：系统会列出所有已创建的项目

### 角色切换

根据你的工作视角，随时切换角色：

```
"切换到总监视角"
  -> 加载 Director Commander
  -> 关注维度：项目组合管理、资源池调度、成本控制
  -> 确认："已切换到总监视角。说说你想看什么？"

"切换到顾问视角"
  -> 加载 Consultant Commander
  -> 关注维度：方法论指导、过程改进、质量审计

"回到项目经理"
  -> 回到 PM Commander（默认角色）
```

### 跨模块上下文传递

在同一会话中依次使用多个模块时，上下文自动传递：

| 上游模块 | 下游模块 | 传递内容 |
|---------|---------|---------|
| 会议纪要 (R02) | 周报 (R01) | 决议和待办作为周报素材，标注 `[来自会议纪要]` |
| 风险评估 (R03) | 周报 (R01) | 新增/升级的风险条目出现在周报"风险与问题"板块 |
| 进度追踪 (R04) | 客户沟通 (R06) | 进度数据填入客户汇报材料 |
| 需求分析 (R07) | 风险评估 (R03) | 需求变更自动触发风险评估 |
| 数据采集 (R09) | 进度追踪 (R04) | 工时和完成情况更新进度快照 |

可以说"不要自动关联"关闭跨模块传递。

## 最佳实践

### 1. 渐进式信息补充

不需要一次填完所有项目信息。系统采用渐进式设计：
- 首次只需项目名称、阶段、类型、行业
- 后续使用时按需补充里程碑、团队成员、客户联系人
- 系统会在需要时主动提示补充

### 2. 善用复合路由

周五写周报时，用"一键周报"替代分步操作。系统会自动按顺序采集数据、抓进度、扫风险、生成周报，上游输出自动注入下游上下文。

### 3. 持续更新项目状态

每次会议后整理纪要、发现风险及时登记、进度有变化及时更新。状态越完整，系统的输出质量越高。

### 4. 根据客户类型选择模板

- 政府/国企项目——正式公文体，使用 `templates/gov/` 模板
- 互联网/SaaS 项目——简洁直白，使用 `templates/saas/` 模板
- 金融项目——合规优先，使用 `templates/finance/` 模板
- 通用场景——使用 `templates/_base/` 模板

### 5. 利用主动洞察

不要关闭主动洞察功能。它会帮你在完成主任务后提醒超期风险、即将到期的里程碑和资源冲突，避免遗漏。

## 常见问题（FAQ）

### Q1: 项目数据存在哪里？会不会丢失？

项目数据存储在本地 `~/.qoderwork/data/Sloth-DeliveryMatrix-Eido/state/` 目录下，以 YAML 文件形式保存。系统在 `_backups/` 目录下自动保留备份。数据不上传云端，完全在你的设备上。

### Q2: 可以同时管理多少个项目？

没有硬性限制。每个项目对应一个独立的状态文件，通过"切换到XX项目"随时切换上下文。

### Q3: 系统无法识别我的指令怎么办？

系统采用三层降级策略：
1. 先根据最近执行的路由推断你是否在补充上一个操作
2. 判断大致方向后，用一句话追问确认
3. 连续两次无法识别时，展示完整能力菜单供选择

你也可以直接说"帮我写周报""看风险"等明确的关键词。

### Q4: 输出内容不准确或需要修改怎么办？

直接告诉系统需要修改的部分："风险那部分改一下""把第三段换成..."。系统只重新输出修改部分，标注 `[已根据反馈调整]`，不会重新生成整份文档。

### Q5: 如何与其他技能配合使用？

安装协同技能后，系统会自动识别并增强工作流。例如安装 Sloth-DeckBuilder-Eido 后，你可以说"把这份汇报做成PPT"，系统会自动将内容传递给 PPT 引擎生成演示文档。不需要额外配置。

|------|------|
| `templates/_base/` | 通用模板（适用于大多数软件交付项目） |
| `templates/gov/` | 政府/国企项目模板（正式公文体） |
| `templates/saas/` | SaaS/互联网项目模板（敏捷风格） |
| `templates/finance/` | 金融项目模板（合规优先） |
